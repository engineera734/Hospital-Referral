# تحديث المرحلة الخامسة

## قبل التشغيل
1. شغّل ملف `sql/phase5_profit_upgrade.sql` داخل Supabase SQL Editor.
2. تأكد من وجود المتغيرات التالية في `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
3. أوقف المشروع ثم شغّل:
   ```bash
   rmdir /s /q .next
   npm run dev
   ```

## ما الذي أضيف في هذه النسخة
- حذف موظف استقبال أو محاسب من صفحة الإدارة.
- جدول نسب ربح الطبيب لكل قسم.
- عرض المستحقات المتراكمة للطبيب بناءً على الحالات المستقبلة فقط.
- تصفية مستحقات الطبيب من صفحة الإدارة أو المحاسبة مع حذف الحالات المستقبلة الخاصة به من النظام.
- تقارير عامة ومخصصة بصيغة HTML قابلة للطباعة والحفظ PDF من المتصفح.
- تعريب الواجهات الأساسية وتحويل الاتجاه إلى RTL.
- إصلاح رفع صورة البروفايل ومشكلة تفريغ حقل الملف.

## ملاحظات
- زر حذف الحالات في صفحة الإدارة يحذف فقط الحالات المستقبلة.
- الحالات أصبحت عمليًا: منتظرة / مستقبلة. التصفية تسجل صرف المستحقات ثم تزيل الحالات المستقبلة للطبيب من الواجهة وقاعدة البيانات.
- إذا كانت صور البروفايل لا تعمل، تأكد أن bucket باسم `profile-images` موجود بالفعل بعد تشغيل ملف SQL.
